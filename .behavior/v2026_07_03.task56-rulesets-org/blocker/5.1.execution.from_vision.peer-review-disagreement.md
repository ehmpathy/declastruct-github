orchestrators*; this logic already lives in its own named function, which is exactly the rule's remedy. the
reviewer's suggested fix ("extract to a named transformer") describes what already exists.

**precedent**: identical code shipped as `getOneRulesetMatchByName` in `getRepoRuleset.ts` (#57, merged).
a change to the org copy would desync it from the repo copy.

- contradict the `has-consistent-conventions` + `arch-smell-scopeleaks` reviewers who approved this parity.

the get-set-gen brief's strict letter lists get/set/gen, but the repo's actual, merged practice uses `del`
for deletions. consistency with the shipped codebase + the "mirror exactly" task instruction outweigh a
strict-letter rename that would leave the org op as the lone outlier.
