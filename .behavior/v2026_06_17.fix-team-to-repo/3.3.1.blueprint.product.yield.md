# blueprint: DeclaredGithubTeamRepoAccess

## summary

add `DeclaredGithubTeamRepoAccess` resource type to grant teams access to repos. follows extant `DeclaredGithubTeamMembership` patterns. enables teams to be environment reviewers.

## github API map

| operation | github endpoint | method |
|-----------|-----------------|--------|
| get | `GET /orgs/{org}/teams/{team_slug}/repos/{owner}/{repo}` | GET |
| set | `PUT /orgs/{org}/teams/{team_slug}/repos/{owner}/{repo}` | PUT |
| delete | `DELETE /orgs/{org}/teams/{team_slug}/repos/{owner}/{repo}` | DELETE |

**notes:**
- GET returns 204 if team has access, 404 if not
- PUT body: `{ permission: 'pull' | 'push' | 'triage' | 'maintain' | 'admin' }`
- DELETE returns 204 on success, 404 if not found (idempotent)

## implementation notes

### repo reference

use `RefByUnique<typeof DeclaredGithubRepo>` to reference repos — same pattern as `DeclaredGithubEnvironment`.

### permission type

use string literal union for compile-time safety:

```ts
permission: 'pull' | 'push' | 'triage' | 'maintain' | 'admin'
```

### skipIf for org admin

integration tests and acceptance apply tests use `given.skipIf(!hasOrgAdmin)` pattern:
- `TEST_ORG_ADMIN=true` enables full lifecycle tests
- CI lacks org admin scope (intentional security constraint)
- apply verified via dogfood (provision/github)

note: this is documented in `.agent/repo=.this/role=any/briefs/howto.test-admin-resources.md` as the standard pattern for admin-scoped resources. not a deferral — a security-driven test strategy.

## repo pattern notes

### del verb follows extant pattern

`delTeamRepoAccess` follows extant repo patterns:
- `delTeam.ts` in domain.operations/team/
- `delTeamMembership.ts` in domain.operations/teamMembership/

the research showed `del*` is the established verb for delete operations in this repo.

### inline transformers follow extant pattern

transformers and communicators are declared inline within orchestrator files per extant pattern:
- `getOneTeamMembership.ts` contains inline `asTeamMembershipOrNull` transformer
- `setTeamMembership.ts` contains inline `asDesiredTeamMembership` transformer
- `setTeam.ts` contains inline `asDesiredTeam` transformer

separate files are used only for shared utilities in `_utils/` (e.g., `isGithubNotFoundError`).

### DAO delegates to domain.operations

the declastruct pattern is: DAOs delegate to domain.operations, not the reverse.
- `DeclaredGithubTeamMembershipDao` delegates to `getOneTeamMembership`, `setTeamMembership`, `delTeamMembership`
- this is directional: domain.operations owns logic, DAOs are thin wrappers for declastruct interface

### skipIf for org admin follows documented pattern

per `.agent/repo=.this/role=any/briefs/howto.test-admin-resources.md`:
- resources that need `admin:org` scope use `given.skipIf(!hasOrgAdmin)`
- CI tokens lack org admin scope for security
- full lifecycle verified via dogfood (provision/github)
