# blocker.1: Missing positive path snapshot for `apply` CLI command

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.verification/rule.require.contract-snapshot-exhaustiveness.md

**locations**:
- src/contract/sdks/__snapshots__/declastruct.acceptance.test.ts.snap

The `apply` CLI command only has snapshots for --help and error cases (empty plan file, invalid syntax, nonexistent plan). There is no snapshot for a successful `apply` execution. This is a complete gap – the positive path is not documented in snapshots.

According to the rule, every CLI command must have snapshots for positive path, negative path, and edge cases. The negative and help cases are present, but the positive path is missing.

The contract output (stdout/stderr) for a successful apply is not snapped, so drift or regressions in the success output will go undetected.


- src/contract/sdks/__snapshots__/declastruct.acceptance.test.ts.snap

The SDK boundary tests only include snapshots for the not-found (negative) path for each resource (repo, config, environment, team, membership). There are no snapshots for the positive path – i.e., when the resource exists and is retrieved successfully.

According to the rule, every SDK method must have snapshots for positive path, negative path, and edge cases. The negative path is covered, but the positive path is completely missing.

Without positive path snapshots, regressions in successful retrievals (shape, content, status) will go undetected.
