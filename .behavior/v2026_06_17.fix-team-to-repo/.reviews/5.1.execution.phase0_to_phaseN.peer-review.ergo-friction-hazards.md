# blocker.2: missing test for DeclaredGithubTeam validation error (secret team with parent)

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.friction-hazards.md

**locations**:
- src/domain.objects/DeclaredGithubTeam.ts

The constructor of DeclaredGithubTeam throws a BadRequestError when a secret team is given a parent. This error path is a friction point for users who accidentally provide a parent for a secret team. However, there is no test in the target files that exercises this validation. Without an acceptance test, the error message may drift or become unclear over time, and reviewers cannot verify the user experience via snapshot.

**snippet**:
```ts
// DeclaredGithubTeam.ts – constructor validation
if (props.privacy === 'secret' && props.parent !== null) {
  throw new BadRequestError('secret teams cannot have a parent', {
    slug: props.slug,
    privacy: props.privacy,
    parent: props.parent,
  });
}
```

# nitpick.1: error messages in castToDeclaredGithubTeamRepoAccess tests use toContain instead of toMatchSnapshot

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.friction-hazards.md

**locations**:
- src/domain.operations/teamRepoAccess/castToDeclaredGithubTeamRepoAccess.test.ts

The tests for castToDeclaredGithubTeamRepoAccess verify that errors are thrown and contain expected substrings, but they do not snapshot the full error messages. The friction rule recommends snapping error messages to enable drift detection. Using toMatchSnapshot would provide a stable record of what the user sees and alert reviewers if the error text changes unintentionally.

**snippet**:
```ts
// castToDeclaredGithubTeamRepoAccess.test.ts – error check without snapshot
then('it should throw because role_name absent', async () => {
  const error = await getError(() =>
    castToDeclaredGithubTeamRepoAccess(input),
  );
  expect(error).toBeInstanceOf(UnexpectedCodePathError);
  expect(error.message).toContain('role_name absent');  // 👎 no snapshot
});
```
