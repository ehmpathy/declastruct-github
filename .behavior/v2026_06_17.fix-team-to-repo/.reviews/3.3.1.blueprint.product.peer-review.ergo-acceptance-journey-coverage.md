# blocker.1: Acceptance tests rely on skipIf for critical apply journeys due to credentials

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/ergonomist/rule.require.acceptance-journey-coverage.md

**locations**:
- .behavior/v2026_06_17.fix-team-to-repo/3.3.1.blueprint.product.yield.md

The blueprint describes using given.skipIf(!hasOrgAdmin) for acceptance apply tests and integration tests. This means the full user journey for applying the DeclaredGithubTeamRepoAccess resource is not executed or snapped in CI. The rule requires every user journey to have acceptance tests with snapshots for positive, negative, and edge cases using real calls. Credential difficulty is explicitly a blocker, not a deferral or excuse. Mentioning dogfood as verification is not a substitute for exhaustive acceptance test coverage in the suite.

**snippet**:
```md

- `TEST_ORG_ADMIN=true` enables full lifecycle tests
- CI lacks org admin scope (intentional security constraint)
- apply verified via dogfood (provision/github)

note: this is documented in `.agent/repo=.this/role=any/briefs/howto.test-admin-resources.md` as the standard pattern for admin-scoped resources. not a deferral — a security-driven test strategy.
```
