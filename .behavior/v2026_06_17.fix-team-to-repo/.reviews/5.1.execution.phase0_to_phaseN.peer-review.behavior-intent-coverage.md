# blocker.1: Integration test for team repo access is skipped due to missing credentials

**rule**: require.external-contract-integration-tests

**locations**:
- src/domain.operations/teamRepoAccess/teamRepoAccess.play.integration.test.ts:27-37

The lifecycle integration test (`teamRepoAccess.play.integration.test.ts`) is conditionally skipped using `given.skipIf(!hasOrgAdmin)` when the environment variable `TEST_ORG_ADMIN` is not set to `true`. This means that in CI (where the variable is absent), no real call to the GitHub API is ever made. The rule `require.external-contract-integration-tests` explicitly states: "lack of credentials is not an excuse. get them, or fail the gate." Furthermore, the verification philosophy (part of the same rule context) says "a skip is a hidden lie. it pretends coverage exists. it passes ci. it lets broken code through." This is a blocker because the external contract (GitHub API for team repo permissions) has no verified integration test in the default CI run. The comment in the file says "skipIf used because CI lacks admin:org scope; apply verified via dogfood", but that is not acceptable per the rule. The provider should either mock the external call in CI with a fallback or ensure credentials are available.
