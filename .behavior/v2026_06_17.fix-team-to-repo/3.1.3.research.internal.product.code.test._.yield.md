## pattern.3: full lifecycle in single test [REUSE]

lifecycle tests cover full add, read, update, remove cycle in one `then` block.

**citation 3:** `src/domain.operations/teamMembership/teamMembership.play.integration.test.ts:30-156`
```ts
when('[t0] lifecycle operations are executed', () => {
  then('should complete add, read, update role, remove', async () => {
    // step 0: create team for membership tests
    const team = await setTeam({ findsert: { ... } }, context);

    // step 1: verify membership does not exist
    const membershipBefore = await getOneTeamMembership(...);
    expect(membershipBefore).toBeNull();

    // step 2: add membership via findsert
    const membershipCreated = await setTeamMembership({ findsert: { ... } }, context);

    // step 3: fetch membership after creation
    const membershipFetched = await getOneTeamMembership(...);

    // step 4: findsert again (idempotent - should not update)
    const membershipFindsertAgain = await setTeamMembership({ findsert: { ... } }, context);

    // step 5: upsert to change role
    const membershipUpserted = await setTeamMembership({ upsert: { ... } }, context);

    // step 6: remove membership
    await delTeamMembership(...);

    // step 7: verify membership is removed
    const membershipAfterDelete = await getOneTeamMembership(...);
    expect(membershipAfterDelete).toBeNull();

    // cleanup: delete team
    await delTeam(...);
  });
});
```

**relevance:** create `teamRepoAccess.play.integration.test.ts` with same lifecycle structure:
1. verify access does not exist
2. add via findsert
3. fetch after creation
4. findsert again (idempotent)
5. upsert to change permission
6. remove access
7. verify access is removed
