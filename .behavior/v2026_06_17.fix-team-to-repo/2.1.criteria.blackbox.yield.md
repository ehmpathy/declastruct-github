# blackbox criteria: DeclaredGithubTeamRepoAccess

## usecase.1 = grant team repo access

### happy path

```
given a team and repo in the same org
  when DeclaredGithubTeamRepoAccess is declared with permission 'push'
    then declastruct plan shows the team will be added to repo
      sothat user can preview the change before apply
    then declastruct apply grants the team push access
      sothat team members can push to the repo
    then declastruct apply is idempotent
      sothat re-apply does not error or duplicate
```

### permission levels

```
given a team and repo in the same org
  when DeclaredGithubTeamRepoAccess is declared with permission 'pull'
    then team members can read the repo
      sothat audit teams can access private repos

  when DeclaredGithubTeamRepoAccess is declared with permission 'push'
    then team members can push to the repo
      sothat contributor teams can commit code

  when DeclaredGithubTeamRepoAccess is declared with permission 'triage'
    then team members can manage issues and PRs
      sothat support teams can triage without code access

  when DeclaredGithubTeamRepoAccess is declared with permission 'maintain'
    then team members can manage repo without admin
      sothat maintainer teams can configure without full control

  when DeclaredGithubTeamRepoAccess is declared with permission 'admin'
    then team members have full repo control
      sothat platform teams can manage repo settings
```

### permission update

```
given a team with 'pull' access to a repo
  when DeclaredGithubTeamRepoAccess is declared with permission 'push'
    then declastruct apply updates permission to 'push'
      sothat declared state becomes source of truth
```

## usecase.2 = enable team as environment reviewer

### prerequisite for environment reviewers

```
given a team with no repo access
  when team is added as environment reviewer
    then declastruct apply fails with 422
      sothat user knows team needs repo access first

given a team with repo access via DeclaredGithubTeamRepoAccess
  when team is added as environment reviewer
    then declastruct apply succeeds
      sothat environment reviewers work as expected
```

## usecase.3 = error cases

### team not found

```
given a team that does not exist
  when DeclaredGithubTeamRepoAccess is declared
    then declastruct apply fails with clear error
      sothat user knows to create team first
```

### repo not found

```
given a repo that does not exist
  when DeclaredGithubTeamRepoAccess is declared
    then declastruct apply fails with clear error
      sothat user knows to create repo first
```

### cross-org team

```
given a team in org 'alpha' and repo in org 'beta'
  when DeclaredGithubTeamRepoAccess is declared
    then declastruct apply fails with 422
      sothat user knows teams can only access repos in same org
```

### invalid permission

```
given a DeclaredGithubTeamRepoAccess declaration
  when permission is not one of pull/push/triage/maintain/admin
    then typescript compile fails
      sothat invalid permissions are caught at build time
```

## usecase.4 = declaration ergonomics

### reference patterns

```
given a declared repo variable
  when DeclaredGithubTeamRepoAccess uses repo ref
    then team/repo are referenced by unique keys
      sothat declarations compose without circular deps

given a team ref with org.login and slug
  when DeclaredGithubTeamRepoAccess uses team ref
    then team is referenced by natural key
      sothat no primary id lookup is needed
```

### declaration order

```
given DeclaredGithubTeamRepoAccess declared before DeclaredGithubTeam
  when declastruct apply runs
    then apply fails if team doesn't exist
      sothat user understands dependency order
```
