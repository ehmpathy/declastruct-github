# nitpick.1: manual try/catch instead of HelpfulError.wrap for github error wrapping

**rule**: .agent/repo=ehmpathy/role=mechanic/briefs/practices/code.prod/pitofsuccess.errors/rule.prefer.helpful-error-wrap.md

**locations**:
- src/domain.operations/team/getOneTeam.ts:42
- src/domain.operations/team/delTeam.ts:42
- src/domain.operations/teamMembership/getOneTeamMembership.ts:45
- src/domain.operations/teamMembership/delTeamMembership.ts:45

The get and del communicators use manual try/catch to handle github API errors: special-case 404 as null/return for idempotency, and wrap others in MalfunctionError with cause. Per the rule, when try/catch is used to make errors more observable (even with allowlisted control-flow cases), use MalfunctionError.wrap (or HelpfulError.wrap) instead for automatic instantiation and maximum observability. The current pattern duplicates what the wrapper does and is inconsistent with set* functions in the same dir that correctly use .wrap().

**snippet**:
```ts
  try {
    const response = await github.teams.getByName({
      org: input.org,
      team_slug: input.slug,
    });
    return castToDeclaredGithubTeam({ data: response.data, org: input.org });
  } catch (error) {
    if (!(error instanceof Error)) throw error;

    // idempotent: return null if team not found
    if (isGithubNotFoundError({ error })) return null;

    throw new MalfunctionError('github.getTeam error', {
      cause: error,
      org: input.org,
      slug: input.slug,
    });
  }
```
