# handoff: org admin credentials required for team lifecycle tests

## summary

i cannot run the full test suite because team operations require org admin permissions (admin:org scope). this is a foreman-only blocker — only the foreman can grant org admin credentials.

## what i tried

1. **ran all tests with available credentials** — types, lint, format, unit tests all pass
2. **verified test structure** — tests use ConstraintError with clear hints, no silent bypasses
3. **confirmed api requirements** — github teams api requires admin:org scope for create/update/delete
4. **checked keyrack** — no org admin key available via `rhx keyrack unlock --owner ehmpath`

## why each approach failed

| approach | outcome |
|----------|---------|
| run with available token | token lacks admin:org scope, teams api returns 404 |
| search for alternative token | keyrack has no org admin key provisioned |
| mock the api | forbidden — integration tests must hit real endpoints |
| skip the tests | forbidden — tests must run, not skip |

## what makes this fundamentally impossible

github teams api requires admin:org scope to:
- create teams (`POST /orgs/{org}/teams`)
- update teams (`PATCH /orgs/{org}/teams/{team_slug}`)
- delete teams (`DELETE /orgs/{org}/teams/{team_slug}`)
- add/remove team members (`PUT/DELETE /orgs/{org}/teams/{team_slug}/memberships/{username}`)

this scope is only available to organization administrators. mechanic credentials operate at repo level, not org level.

## is this truly a "foreman possesses the key" situation?

**yes.** org admin tokens grant write access to all organization resources. this is a security boundary that mechanic credentials cannot cross by design.

## what foreman needs to do

1. provision a github token with `admin:org` scope
2. store it in keyrack as `ehmpathy.org.admin` or similar
3. run tests with `TEST_ORG_ADMIN=true`:

```bash
export TEST_ORG_ADMIN=true
export GITHUB_TOKEN=<org-admin-token>
npm run test:integration
npm run test:acceptance
```
