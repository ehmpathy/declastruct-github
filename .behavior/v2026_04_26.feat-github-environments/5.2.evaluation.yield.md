# evaluation: DeclaredGithubEnvironment

## summary (as implemented)

implemented `DeclaredGithubEnvironment` domain object with get/set/del operations to enable declarative management of GitHub deployment environments:

- domain object with repo + name as unique key (per extant patterns)
- reviewers accept usernames/team slugs, numeric ID lookup is internal
- branch policy supports: all branches (null), protected only, or custom patterns
- atomic operations: if branch pattern sync fails, environment is rolled back
- ergonomic errors: include problematic username/pattern in messages
- **additional**: extracted `syncDeploymentBranchPolicies` to separate file in peer review (satisfies decode-friction rules)
