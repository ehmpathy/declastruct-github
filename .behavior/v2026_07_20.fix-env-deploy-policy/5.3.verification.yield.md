## defect caught + fixed at this gate

the acceptance CLI `plan` (real serialize path) surfaced a bug the unit + integration suites did
not: `customPatterns` was a raw object array on a generic `DomainLiteral`, which domain-objects'
`serialize` rejects (`DomainObjectNotSafeToManipulateError`). fix: `deploymentBranchPolicy` is now a
named `DeclaredGithubEnvironmentDeploymentBranchPolicy` DomainLiteral whose `static nested` declares
`customPatterns` (each item a flat `DomainLiteral`) — mirrors `DeclaredGithubRepoRuleset.bypassActors`.
this is a test-covered repair: the acceptance suite proves the fix (plan now succeeds).
