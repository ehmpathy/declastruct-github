# blocker — 5.1.execution.from_vision

## what blocks me

peer review converged to **one** open blocker, flagged by 4 reviewers across i004–i007 and,
per **both** terminal l3 enroll reviewers, a decision that "belongs to a human, not silently
to CI":

the acceptance suite gates the CLI `declastruct apply` verification behind a silent skip:

```ts
// src/contract/sdks/declastruct.acceptance.test.ts
const hasOrgAdmin = process.env.TEST_ORG_ADMIN === 'true';
when.skipIf(!hasOrgAdmin)('[t1] plan is applied via declastruct CLI', () => { ... });
```

per this repo's own `rule.forbid.failhide`, `when.skipIf` is **not** an acceptable path for an
absent resource — it should fail loud. this is a real rule violation.

**why it is a genuine wall, not a robot fix — three linked constraints:**

1. **two repo briefs conflict.** the skipped block bundles team + membership rows that need
   `admin:org` scope, which CI lacks *by design* (`howto.test-admin-resources`: admin resources
   are dogfooded, not CI-tested). so `rule.forbid.failhide` (never skip) and
   `howto.test-admin-resources` (admin resources cannot run in CI) directly contradict for this
   block. a reconciliation of the two is a scope/architecture call above the blueprint's authority.

2. **the correct fix is credential-blocked.** the right structural fix (a split of the fixture into
   a repo-scoped set that applies unconditionally in CI + an admin-scoped set that stays gated)
   needs a real `GITHUB_TOKEN` to author and verify the new `apply` path. per
   `howto.supply-github-token-for-tests`, an absent token is explicitly "a robot blocker, not a
   robot fix — only a human can fill a credential; robots must surface and wait, never fake,
   skip, or work around it." an unverifiable `apply` that leans on CI to catch my errors *is* a
   workaround of the absent token.

3. **the small fix reddens CI.** a swap of `skipIf` → `ConstraintError.throw` converts silent-skip
   to loud-fail per the rule's own prescription, but since CI genuinely lacks `admin:org`, it
   would turn CI red on every run.
