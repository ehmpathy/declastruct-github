# blocker.1: Mutation by sorting arrays in place violates immutability rule

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.maintenance-hazards.md

**locations**:
- src/domain.operations/repoRuleset/castToDeclaredGithubRepoRuleset.ts:36
- src/domain.operations/repoRuleset/castToDeclaredGithubRepoRuleset.ts:55

The functions castToBypassActors and castToRules use .sort() which mutates the array in place. This violates the immutability requirement under maintenance hazards: variables should not be mutated; use spread or .slice() to create a new array before sorting to avoid side effects.

Example of violation in castToBypassActors: the actors array is built via map and then sorted with actors.sort(), mutating the original array. Similarly in castToRules, rules.sort() mutates the array.

**snippet**:
```ts
const actors = (input.bypassActors ?? []).map((actor) => ({
  actorId: actor.actor_id ?? null,
  actorType: actor.actor_type as DeclaredGithubRepoRulesetBypassActor['actorType'],
  bypassMode: (actor.bypass_mode as DeclaredGithubRepoRulesetBypassActor['bypassMode']) ?? 'always',
}));

// sort by actorType then actorId for a canonical order
return actors.sort((a, b) => {
  if (a.actorType !== b.actorType) return a.actorType.localeCompare(b.actorType);
  return (a.actorId ?? -1) - (b.actorId ?? -1);
});
```

# nitpick.1: Unnecessary type assertion on return value
