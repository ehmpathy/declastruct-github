## design decisions made during build (refinements to the vision)

1. **`get.byPrimary` is `undefined`, not "wire both".** the vision Q2 leaned "wire both", but
   github's get-by-id endpoint also needs owner+repo, which `RefByPrimary` (id only) does not
   carry. so by-unique (list+filter by name → get-by-id) is the only viable lookup. matches the
   environment/branch-protection precedent of `byPrimary: undefined`.

2. **value objects flattened for declastruct serialize (acceptance-test-driven).** the acceptance
   test caught a real defect: declastruct's `serialize` requires every object-valued field to be a
   declared DomainObject, and a `DomainLiteral` is only safe when its fields are flat
   (primitives / primitive-arrays). so:
   - `conditions` is `{ refNameInclude, refNameExclude }` (flat), not `{ refName: { include, exclude } }`
   - `rules` carries only `{ type }`; rule-type-specific `parameters` are out of v1 scope (all
     supported rule types are parameterless except `update`'s one optional flag)
   - `bypassActors` is `{ actorId, actorType, bypassMode }[]` (already flat)
   the cast translates github's nested `ref_name` to/from the flat domain shape.

3. **canonical sort in the cast** keeps `plan` diffs stable: rules sorted by `type`, bypass actors
   by `actorType`/`actorId` — so a declared order never false-drifts against github's returned order.
